Lösen einer Aufgabe

Beispieleingabe

Die Beispieleingabe gibt Ihnen eine syntaktisch korrekte Eingabe vor, die jedoch im allgemeinen keine korrekte Lösung Ihrer gegebenen Aufgabeninstanz ist.

Letzte Eingabe

Dieser Knopf lädt grundsätzlich die Eingabe, die Sie zuletzt abgeschickt haben, unabhängig davon, ob diese Eingabe syntaktisch oder inhaltlich korrekt war.

Letzte korrekte Lösung

Mit dieser Schaltfläche laden Sie die letzte Eingabe, die korrekt war, d.h. einen Punktestand verschieden von Null erzielte.

Beste Lösung

Ob sich die beste Lösung durch einen besonders niedrigen positiven oder einen möglichst hohen Punktwert auszeichnet, hängt vom speziellen Aufgabentyp ab. Welche Variante für einen speziellen Aufgabentyp gilt, ergibt sich meist direkt aus der Aufgabenstellung.

Topliste anzeigen

Die Topliste zeigt Ihnen in einem anonymisierten Überblick, welche Punkte Ihre Kommilitonen erzielt haben. Dabei werden nur solche Studenten erfasst, die bereits mindestens eine korrekte Lösung eingesandt haben. Ihre eigene Position in der Topliste ist farbig gekennzeichnet, so dass Sie einschätzen können, wo Sie im Vergleich liegen.

Topliste

Aus Sicht der Betreuer

Es wird eine Tabelle mit allen Studenten angezeigt. In dieser Tabelle sind pro Aufgabe die erreichten Punkte und die Punkte - berechnet nach den Scorepoints - zu sehen. Außerdem wird angezeigt ob der Student bestanden hat. Am Ende werden diese Informationen zusammengerechnet.

Aus Sicht der Studenten

Hier wird eine Tabelle mit den erreichten Punkten angezeigt, die bereits mit den Scorepoints verrechnet wurden. Namen oder Matrikelnummern sind nicht zu sehen. Die Punkte des Studenten sind hervorgehoben und zeigen im Falle, dass die Aufgabe bestanden ist, ein grünes Häkchen, sonst einen rot durchgestrichenen Kreis.

Konfiguration des Aufgabenknotens

Verbindungskonfiguration

Hier können Sie für diesen autotool-Baustein die zu nutzende Server-Verbindung wählen. Wählen Sie dazu eine bestehende Verbindung aus der ausklappbaren Auswahlliste aus. Ihre Änderungen werden mit einem Klick auf "Speichern" übernommen.

Grundsätzlich sollte immer die neueste Version des autotool-Servers verwendet werden. Die als Nightly markierte Version dient nur zum Testen und sollte nicht im Produktiveinsatz verwendet werden.

Wahl des Aufgabentyps

Sie können über einen Klick auf "Typ wählen" den Aufgabentyp der autotool-Aufgabe bestimmen. Wählen Sie dazu einen Typeintrag im sich öffnenden Übersichtsbaum aller Aufgabentypen. Wenn sich nach dem Klick das Fenster mit Übersichtsbaum schließt, wurde der Typ erfolgreich gewählt und wird anschließend im Kasten "Aufgabentyp" angezeigt.

Der gerade beschriebene Schritt ist der erste bei der Konfiguration einer neuen Aufgabe. Wenn der "Typ wählen"-Button grau hinterlegt und gesperrt ist, kann es dafür zwei Gründe geben: Zum Einen ist für diesen Aufgabenbaustein noch keine Serververbindung hinterlegt. Dann wählen Sie zunächst unter "Verbindung" einen autotool-Server, bevor Sie den Aufgabentyp bestimmen können. Zum Anderen ist es möglich, dass die aktuelle Aufgabe bereits von Studenten bearbeitet wurde. Eine Änderung des Aufgabentyps ist dann nicht mehr möglich.

Konfiguration der Aufgabe

Sie können eine im System vorhandene Aufgabenkonfiguration für den gewählten Aufgabentyp laden. Klicken Sie dazu auf "Auswahl öffnen". Alternativ können Sie die aktuelle Aufgabenkonfiguration bearbeiten. Wenn Sie Änderungen vornehmen - dies betrifft sowohl die Aufgabenkonfiguration als auch den Kommentar für Autoren und die Studentenbeschreibung der Aufgabe -, dann werden diese nach einem Klick auf "Übernehmen" gespeichert.

Der Dokumentationslink beschreibt den Haskell-Typ, den die Konfiguration der Aufgabe besitzen muss.

Testen der Aufgabe

An dieser Stelle lässt sich die Aufgabenkonfiguration aus der Sicht eines Studenten für eine zufällig generierte Aufgabeninstanz testen. Genau diese Ansicht haben die Studierenden später zur Lösung der Aufgabe. Wenn Sie eine Lösung eintragen und absenden, erhalten Sie ein entsprechendes Feedback vom autotool-Server. Auf diese Weise können Sie Fehler oder Unwegbarkeiten in der Aufgabenkonfiguration durch das Testen von Lösungen finden.

Da Sie die Konfiguration der Aufgabe später - nachdem mindestens ein Student die Aufgabe bereits betrachtet hat - nicht mehr verändern können, sollten Sie die Aufgabe vor der Veröffentlichung gründlich testen.

Gliederungsabschnitte

Hier können Sie die Bearbeitung der Aufgabe durch die Studenten in Abschnitte gliedern, welche jeweils markieren, wie lange eine einzelne konkrete Aufgabe für einen Studierenden gültig ist. Nach Ablauf der im Gliederungsabschnitt definierten Bedingungen wird für den Studierenden eine neue Aufgabe ausgewürfelt.

In einem Gliederungsabschnitt lassen sich nach einem Klick auf "Abschnitt erstellen" drei Dinge beschränken: das Enddatum des Abschnitts, die Zeitdauer relativ zur Erzeugung der Aufgabe dieses Abschnitts sowie die Versuchsanzahl für eine Aufgabe in diesem Abschnitt. Darüber hinaus können Sie festlegen, wo Sie den neuen Abschnitt einfügen wollen - am Anfang der Liste oder nach einem bereits bestehenden Abschnitt. Wenn Sie den Abschnitt mit einem Klick auf "Speichern" übernehmen, wird geprüft, ob sich der Abschnitt mit anderen überschneidet oder gegebenenfalls die Gesamtbearbeitungszeit der Aufgabe überschritten wird. In diesem Fall erhalten Sie einen Hinweis und können Ihre Eingaben korrigieren.

Eine Übersichtstabelle über alle vorhandenen Abschnitte wird angezeigt, sobald Abschnitte zu dieser Aufgabe vorhanden sind. Diese können editiert oder gelöscht werden, solange noch kein Student mit der Bearbeitung der Aufgabe begonnen hat. Erreicht ein Student das Ende des letzten Abschnitts, so wird dieser nachfolgend bis zum Bearbeitungsende wiederholt, es sei denn, ein Enddatum wurde angegeben.

Bewertungszeitraum und Topliste

Beginn und Ende des Bewertungszeitraumes

Hier können Sie den Zeitraum festlegen, in welchem Lösungen der Studenten für diese Aufgabe in die Bewertung einfließen sollen. Der Bewertungszeitraum ist für die Studenten bindend, d.h. jede Lösung vor Beginn oder nach Ende des Zeitraums geht nicht in die Bewertung ein.

Den Startzeitpunkt können Sie beliebig wählen, solange er vor dem Ende des Bewertungszeitraumes liegt. Der Endzeitpunkt der Aufgabe darf nicht in der Vergangenheit liegen und muss zeitlich hinter dem Beginn der Bewertung gewählt werden.

Scorepoints

An dieser Stelle werden die Scorepoints eingetragen. Bei der Berechnung der Top-Liste für diese Aufgabe werden die erreichten Punkte der besten Studenten durch diese Punkte ersetzt. Sie bekommen gegenüber den übrigen Studenten für bessere Leistungen einen Bonus.

Im- und Export von Aufgaben

Mit einem Klick auf "Aufgabe importieren" können Sie eine Aufgabenkonfiguration, welche Sie lokal auf Ihrem Rechner gespeichert haben, in den Aufgabenknoten importieren. Wählen Sie dazu im sich öffnenden Dialog über "Datei wählen" die .zip-Datei auf Ihrem Rechner aus, welche die Aufgabe enthält. Anschließend wird durch den "Hochladen"-Button der Import-Vorgang ausgelöst. Falls ein Fehler auftritt, kann es hilfreich sein, die zu importierende Aufgabe neu zu exportieren, um Fehler in der gespeicherten Datei auszuschließen. Der "Aufgabe importieren"-Button ist gesperrt, falls Studenten die gegenwärtige Aufgabe bereits bearbeiten.

Über den Button "Aufgabe exportieren" lässt sich die vorliegende Konfiguration dieses Aufgabenknotens exportieren. Dazu muss bereits eine Aufgabe konfiguriert sein. Nach dem Klick öffnet sich der Download-Dialog Ihres Browsers mit einer .zip-Datei, die die Aufgabenkonfiguration des aktuellen Knotens enthält. Bitte beachten Sie: Nehmen Sie keine manuellen Änderungen an dieser Datei oder ihrem Inhalt vor, um den Import-Vorgang nicht zu gefährden!

Verwaltung der Serververbindungen

Nicht vollständig implementiert!

In diesem Bereich können Sie die bestehenden Verbindungen zu autotool-Servern verwalten. Klicken Sie dazu auf "Bearbeiten". Es öffnet sich eine Übersichtstabelle aller Verbindungen, die im System registriert sind. Inaktive Verbindungen können Sie mit einem Klick auf "Aktivieren" aktiv schalten. Für nicht verwendete Verbindungen besteht eine Lösch-Option. Alle Verbindungen können durch einen Klick auf "Bearbeiten" editiert werden. Im sich öffnenden Dialog lässt sich dann sowohl der Name als auch die konkrete Server-Adresse editieren.
